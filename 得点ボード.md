得点ボード

要件
Webサイト上で得点を入力して反映できるツール
反映先は
別ページと作成した
OBSテロップオーバレイページ


## 技術スタック
- フロントエンド：HTML / CSS / Vanilla JS、socket.io-client
- バックエンド：Node.js (Express)、socket.io、JSONファイルストレージ（ファイルI/O）、sharp（画像変換）*、dotenv
  - *sharp は現在必須ではなく、OBS用の画像生成機能を追加する場合に使用
- サーバ：Xserver レンタルサーバ（Apache リバースプロキシ経由、ポート3000）
- プロセス管理：nohup + cron（pm2 が使える環境では pm2 を推奨）
- テスト：Jest + supertest

## データ設計（JSONストレージ）
- 保存先ファイル: `data/teams.json`（or `./teams.json`）
- 形式:
  ```json
  [
    { "id": 1, "name": "チーム1", "score": 0 },
    ...
    { "id": 10, "name": "チーム10", "score": 0 }
  ]
  ```
- アクセスパターン:
  - アプリ起動時にファイルを読み込みキャッシュ
  - 更新時はメモリ更新→ファイル書き込み（atomic: tmpファイル経由、rename）
  - エラー時はバックアップファイル(`teams.bak.json`)を使用して復旧
- 複数プロセス同時書きの懸念:
  - 単一Nodeプロセス運用を前提
  - 必要なら`flock`相当のロック機構を追加

## 仕様補足
- JSONファイルストレージを採用して構成を簡素化
- `fs.promises.readFile`, `writeFile`を利用
- 一時的にスコアの永続化が不要な場合はメモリのみ実装も可能（再起動でリセット）

## 目的
- Web画面で得点を入力／更新し、リアルタイムで表示（OBS用オーバーレイ含む）する管理ツール
- 10チーム（5列 x 2行）で固定レイアウト
- 管理者が得点操作し、結果を別ページ（OBSテロップオーバーレイページ）に反映

## 機能
### 1) チームデータ管理
- 10チーム固定
- 各チーム項目
  - `team_id` (1〜10)
  - `team_name`
  - `score`（数値、初期0）

### 2) 得点入力/更新
- 管理画面
  - チームごと +1/-1 ボタン
  - 数値直接更新
  - リセット（全チーム0）
- 更新時
  - JSONストレージ反映
  - socket.io通知

### 3) 表示ページ (得点表示 & OBSオーバーレイ)
- 公開ページ
  - 5×2グリッド表示
  - 画面サイズ: 1920x1080 のうち下1/3領域（高さ360px）に収まるレイアウト
  - エリアは上下左右に余白を確保（上下: 20px, 左右: 30px程度）
  - 見出し: 背景 #e50000, 文字白, 黒縁
  - 得点最大4桁(0~9999だけど一応それ以上も制限はしない)で、余裕を持たせたフォントサイズ
  - スコアボックスは固定幅（例: 220px）で桁数に余裕
- リアルタイム同期 via socket.io
- 再接続時は最新情報を再取得

### 4) リアルタイム同期
- 管理画面からイベント送信
- 表示ページがイベント受信し更新
- 失敗時はAPIフェールオーバーでfetch

## API設計
- `GET /api/teams` → 全チーム取得
- `POST /api/teams/:id/score` → `{ "score": 12 }`または`{ "delta": 1 }`
- `POST /api/teams/reset-all` → 全リセット

## 非機能要件
- 更新遅延 < 1秒
- 99%可用性を目指す
- 1分あたりリクエスト1000未満
- 障害時はエラーメッセージ表示

## テスト
- Jest + supertest
- socket.ioイベントテスト

## 運用
- Xserver + Apacheリバースプロキシ（Port3000）
- pm2を推奨（ユースケースによってnohup + cronでも可）
